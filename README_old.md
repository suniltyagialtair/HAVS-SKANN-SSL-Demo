# SKANN-SSL V5 — Vessel Acoustic Demo

Standalone demo tool for the SKANN-SSL V5 passive acoustic vessel detection system.  
**Oravont Systems LLP** for **Altair Infrasec Pvt Ltd**.

---

## Two operating modes

| Mode | Input | When | Outputs |
|---|---|---|---|
| **Timeline** | Any-length WAV (e.g. 60-min session) | WAV > 65 s | `timeline_*.png` + `radar_*.png` per event + `summary_*.csv` |
| **Radar** | Exactly 60-second WAV (one event clip) | WAV ≤ 65 s | Single `radar_*.png` |

Mode is **auto-selected** by WAV duration. Force with `--mode timeline` or `--mode radar`.

## Usage

```bash
# 60-minute session → auto timeline mode
python skann_demo.py \
    --wav session_001.wav \
    --bundle SKANN_SSL_V5_Production_Bundle.joblib

# 60-second event clip → auto radar mode
python skann_demo.py \
    --wav event_clip.wav \
    --bundle SKANN_SSL_V5_Production_Bundle.joblib

# Override output directory
python skann_demo.py \
    --wav session.wav \
    --bundle bundle.joblib \
    --out ./results/day1

# More sensitive detection (default 2.0 — lower = more events detected)
python skann_demo.py \
    --wav session.wav \
    --bundle bundle.joblib \
    --rms-thresh 1.5

# Force a specific mode
python skann_demo.py --wav session.wav --bundle bundle.joblib --mode timeline
python skann_demo.py --wav event.wav   --bundle bundle.joblib --mode radar
```

### All arguments

| Argument | Required | Default | Description |
|---|---|---|---|
| `--wav` | Yes | — | Path to 512 Hz WAV file |
| `--bundle` | Yes | — | Path to `SKANN_SSL_V5_Production_Bundle.joblib` |
| `--out` | No | `./skann_results` | Output directory |
| `--mode` | No | `auto` | `auto` / `timeline` / `radar` |
| `--rms-thresh` | No | `2.0` | RMS detection threshold multiplier |

## Output files

### Timeline mode (e.g. `session_042.wav` with 3 events)

```
skann_results/
    timeline_session_042.png      ← Full session: RMS trace + event bands + event table
    radar_session_042_001.png     ← Radar plot, event 1
    radar_session_042_002.png     ← Radar plot, event 2
    radar_session_042_003.png     ← Radar plot, event 3
    summary_session_042.csv       ← Event table with all probabilities
```

**Timeline PNG** contains two panels:
- **Top**: RMS energy trace across the full session, coloured bands where vessel events are detected, cluster label at the top of each band
- **Bottom**: Event table with start/end time (minutes), duration, cluster, confidence, and per-cluster probabilities

### Radar mode (e.g. `event_clip.wav`)

```
skann_results/
    radar_event_clip.png          ← Spider plot: probability across C0–C4
```

## Clip length guidance

| Use case | Recommended input | Mode |
|---|---|---|
| Live session monitoring | 60-minute WAV | Timeline (auto) |
| Inspect a specific transit | 60-second pre-clipped WAV | Radar (auto) |
| Short test clip | Any duration ≤ 65 s | Radar (auto) |
| Custom | Use `--mode` to force | Either |

**Important**: The V5 encoder window is exactly 60 seconds (30,720 samples at 512 Hz). For radar mode, provide a 60-second clip for best results. Shorter clips are zero-padded; longer clips are truncated.

## Detection tuning

If no events are detected in a session:
```bash
python skann_demo.py --wav session.wav --bundle bundle.joblib --rms-thresh 1.5
```

If too many false detections:
```bash
python skann_demo.py --wav session.wav --bundle bundle.joblib --rms-thresh 2.5
```

The detector uses a **two-pass global baseline**: the 15th-percentile RMS across the session is computed first, then events are detected at `baseline × rms-thresh`. An 8-second sustain gate and 5-second gap-bridging filter are applied.

---

## Known limitations (V5)

- A **51–53 Hz infrastructure tone** at the NUWR Goa site reduces inter-cluster separation. All embeddings share a common tonal component; cluster probabilities may be compressed toward 0.20 across clusters rather than peaking sharply. This is a known V5 limitation — addressed in V6.
- Sessions 145–176 contain data quality issues. Embeddings from these sessions may be less reliable.
- Cluster labels (C0–C4) describe acoustic character, not named vessel types.

## Architecture

```
Input WAV (512 Hz)
    │
    ▼
RMS Event Detector
  2 s frames, 1 s hop
  Global 15th-percentile baseline
  8 s sustain gate, 5 s gap bridging
    │
    ▼ (one branch per detected event)
Tensor Extraction
  60-s windows, 75% overlap, up to 50 windows per event
  Unit-RMS normalised
    │
    ▼
SK Filterbank (1D)
  8 × Conv1d kernels: (7,15,31,63,127,255,511,1023)
  64 filters/branch, GELU + GroupNorm
  SK attention fusion
  Frequency coverage: 0.50–73.1 Hz @ 512 Hz
    │
    ▼
2D ResNet Backbone
  L1(1×1) → L2(1×2) → L3(1×2) → L4(2×1) → L5(2×2) → GAP
  → 512-dim h embedding
    │
    ▼
Cosine similarity to K-Means centroids (C0–C4)
Softmax with temperature=8
    │
    ▼
Radar plot (60-s clip) / Timeline + per-event radars (session)
```
